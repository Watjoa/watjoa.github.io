First steps and tips

  • Always use R on your local computer and not via athena

  • Always download the latest version of R and Rstudio

  • Running a code can be done by ctrl + enter (windows) or cmd + enter (mac)

  • Always direct your R code to a specific file in which you want to use a dataset and save all your files. You can do this by setwd("/Users/joachimwaterschoot/Downloads/R") (example for Downloads folder on mac)

  • If you have an error, always google this one. A lost of fora provide you a clear answer on what is going on

  • A warning (= you can continue) is not an error (= nothing happpened)

Show code 
lapply(packages, require, character.only = TRUE)

Refer to your folder where you have downloaded all packages and load them all at once

Show code 
lapply(packages, require, character.only = TRUE, lib.loc=packages_folder)

The loading of packages has to be done each time you open R. Before you are able to load (and use) them, you have to download these on your computer. You can refer to a specific file wherein you save the packages (here the Downloads folder on a mac as example).

Show code 
install.packages(packages,lib=packages_folder)

! Attention: some of the packages are not available in cran and cannot be downloaded by the function above. When you see an error saying that the package cannot be installed, please check google. Most of the times, you have to download the package from github using the devtools package (e.g. devtools::install_github("author/package") )

               idvar = "ppnr",

                 var_1=c('var_1_time1','var_1_time2','var_1_time3'),
                 var_2=c('var_2_time1','var_2_time2','var_2_time3'),
                 var_3=c('var_3_time1','var_3_time2','var_3_time3')
               ),
               direction="long",
               v.names = c(

               ),
               sep="_")

In R, there is the possibility to calculate scale scores for all cases and multiple variables with the same response scale at the same time. Therefore, we need the package psych.
First, a list-object is made including all names of the variables (see var1 and var2). Variables are separated by ,. Important, the minimum and maximum values for these items are the same and need to be described.

Maybe, before starting, an items has to be recoded:

Show code 
data$item.r <- as.factor(recode(data$item, '1'='5', '2'='4','3'='3','4'='2','5'='1'))
Show code 
library(psych)

# assign the items to the variable or scale score
keys.list <- list(
  var1 = c("item4","item7","item11","item15"),
  var2 = c("item5","item6","item7","item8")
) 

keys <- make.keys(data,keys.list) 

# describe the minimum and maximum values of the items
scores <- scoreItems(keys,data,
                       min=1,max=5,
                       missing=TRUE) 

# check the first 6 scale scores in your dataset
head(scores$scores)

Missing values

The first choice is to impute the missing data based on the median of the variable. This is used as default.
Show code 
scores <- scoreItems(keys,data,
                       min=1,max=5,
                       impute="median")
The second choice is to formulate specific variables as missing value NA when a specific number of items is not completed.
Show code 
Percentage <- 80 #if more than 80% of the items is missing, then the scale score becomes a missing values

scores <- scoreItems(keys,data,
                       min=1,max=5,
                       impute="none") # no imputation is done here

colnames(scores$missing) <- paste0('missing_', colnames(scores$missing))

# calculate the percentage of missing values
percNA <- as.data.frame((scores$missing/scores$n.items)*100)

NAcells <- which(percNA > Percentage)

scores$scores[NAcells] <- NA

Save dataset

You can export your dataset to multiple file types

Show code 
# SPSS
write_sav(data, "FILENAME.sav")

# CSV
write_csv(data,'FILENAME.csv')

# EXCEL
write_xlsx(data,'FILENAME.xlsx')

# RDS
saveRDS(data, "FILENAME.rds")
